<배운것>

-용어-
Issue : 내가 할 작업, 버그 리포트, 기능 추가 등으로 쓰인다.
Branch : 특정 commit 에서 갈라져나와 작업할 수 있게 하는 것. 기능별로 이름 지정 가능. 작업할 브랜치로 바꾸는 것을 체크아웃(checkout)이라고 함.
Merge : commit들을 다른 branch로 합치는 것.
Merge conflict : Merge하는 과정에서 같은 파일에 동일한 부분이 수정된 것이 발견되는 것



-실행시 유의사항-
1. 협업할 때는 issue 정하기 - 각자 업무 Branch로 만들기 - 각자의 프로젝트 merge
2. Merge conflict 발생 시, git에서 수정 필요한 부분은 안내해주기 때문에 참고하여 수정하면 됨
3. Push 끝난 Branch는 바로 삭제해주는 것이 편리함.
